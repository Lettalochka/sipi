# Практическая работа 9. Организация разработки

## Цель

Организовать процесс разработки выбранного программного проекта: выбрать
методологию управления разработкой, создать удаленный Git-репозиторий и описать
инструменты разработки.

## Выбранный проект

В качестве проекта рассматривается `Legal Store` - веб-приложение интернет-магазина
цифровых товаров и лицензионных ключей. Основные функции системы:

- регистрация и авторизация пользователей;
- просмотр каталога товаров;
- фильтрация, поиск и сортировка товаров;
- добавление товаров в корзину;
- оформление заказа;
- интеграция с платежным и товарным провайдером Digiseller;
- автоматическая выдача цифрового товара;
- административная панель управления товарами, категориями и заказами.

## Выбранная методология

Для проекта выбрана гибкая итерационно-инкрементная методология на основе Scrum
с элементами Kanban.

Scrum подходит проекту, потому что требования к интернет-магазину могут меняться
по мере разработки: могут добавляться новые сценарии оплаты, способы выдачи
товара, настройки каталога, языковые версии и административные функции. Разработка
короткими спринтами позволяет регулярно получать рабочую версию продукта и
проверять ее на практике.

Kanban используется как дополнительный подход для визуализации текущих задач:
карточки задач проходят стадии `Backlog`, `To Do`, `In Progress`, `Review`,
`Testing`, `Done`. Это удобно для отслеживания состояния задач без усложнения
процесса.

## Организация процесса

Разработка ведется итерациями длительностью 1-2 недели. В начале итерации
формируется список задач, в конце выполняется проверка результата и корректировка
дальнейшего плана.

Основные роли:

- заказчик/владелец продукта - определяет приоритеты функций;
- разработчик - реализует клиентскую и серверную часть;
- тестировщик - проверяет пользовательские сценарии;
- администратор проекта - контролирует сроки и состояние задач.

Основные этапы итерации:

- анализ требований;
- проектирование решения;
- реализация функций;
- тестирование;
- исправление ошибок;
- фиксация результата в Git;
- подготовка документации.

## Удаленный репозиторий

Для хранения материалов используется GitHub-репозиторий:

`https://github.com/Lettalochka/sipi`

В репозитории размещаются отчеты, документация и материалы по практическим
работам. Git используется для контроля версий и фиксации изменений.

## Инструменты разработки

Для разработки проекта используются следующие инструменты:

- `Visual Studio Code` - редактор кода;
- `Git` - система контроля версий;
- `GitHub` - удаленное хранилище репозитория;
- `Node.js` и `npm` - среда выполнения и менеджер пакетов;
- `Next.js` - фреймворк веб-приложения;
- `React` - библиотека для разработки интерфейса;
- `TypeScript` - язык разработки с типизацией;
- `Prisma` - ORM для работы с базой данных;
- `PostgreSQL` - реляционная база данных;
- `DBeaver` - инструмент просмотра и администрирования базы данных;
- `TypeDoc` - генератор документации разработчика;
- `GitHub Wiki`/Markdown - формат пользовательской документации.

## Вывод

Выбранная организация разработки позволяет постепенно развивать проект,
контролировать изменения через Git и поддерживать документацию в актуальном
состоянии. Итерационный подход снижает риск ошибок и дает возможность регулярно
получать рабочие версии программного продукта.
