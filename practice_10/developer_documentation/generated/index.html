    winkey

    Modern e-commerce for legal digital goods (license keys, digital products) with:

    • Next.js (App Router) + TypeScript + Tailwind + shadcn-style components
    • Prisma + PostgreSQL
    • NextAuth (Credentials)
    • Digiseller checkout (oplata.info) + callback-confirmed order lifecycle
    • Automatic fulfillment pipeline with retries (Digiseller + MOCK fallback), encrypted delivery payloads
    • RU/EN UI toggle persisted in cookie
      • Prices follow locale: RU=RUB, EN=USD

    Features (1:1 flows)

    • Home (/): hero search, trending/popular/recent sections
    • Shop (/shop): filtering, sorting, pagination, skeleton loading
    • Product (/product/[slug]): details, add-to-cart, trust blocks
    • Cart (/cart): update qty/remove, totals, empty state
    • Auth (/login, /register): email/password
    • Checkout (/checkout): Digiseller (oplata.info) redirect flow
    • Order details (/order/[id]): secure access (user session or guest access cookie), delivery content decrypted server-side
    • Admin (/admin): products/categories CRUD, orders + fulfillment logs + manual retry

    Getting Started

    1) Configure env

    Copy .env.example → .env and fill at least:

    • DATABASE_URL
    • NEXTAUTH_URL, NEXTAUTH_SECRET (or reuse SESSION_SECRET)
    • DELIVERY_ENCRYPTION_KEY
    • Digiseller keys (DIGISELLER_SELLER_ID, DIGISELLER_API_KEY)
    • JOBS_SECRET (protects the fulfillment job endpoint)
    • (Optional) DIGISELLER_AUTO_SYNC (auto-sync catalog on server start)

    2) Start Postgres

    docker compose up -d

    This repo maps Postgres to localhost:54329 (to avoid conflicts with an existing local Postgres on 5432).

    3) Migrate + seed

    npm run prisma:migrate
    npm run seed

    Seed creates an admin user (defaults shown in .env.example).

    4) Run dev server

    npm run dev

    Open http://localhost:3000

    Fulfillment worker (Digiseller)

    How it works

    • Digiseller callback marks order FULFILLING and creates a Fulfillment row.
    • A job endpoint processes pending fulfillments, stores encrypted delivery payload, and marks order DELIVERED.

    Run jobs

    The worker is a signed endpoint:

    • POST /api/jobs/fulfill with header x-jobs-secret: $JOBS_SECRET
    • POST /api/jobs/sync-digiseller with header x-jobs-secret: $JOBS_SECRET (refresh USD/RUB prices from Digiseller)
      • Add header x-sync-mode: catalog to also refresh titles/descriptions/images/categories from Digiseller

    Example:

    curl -X POST http://localhost:3000/api/jobs/fulfill -H "x-jobs-secret: $JOBS_SECRET"
    curl -X POST http://localhost:3000/api/jobs/sync-digiseller -H "x-jobs-secret: $JOBS_SECRET"
    curl -X POST http://localhost:3000/api/jobs/sync-digiseller -H "x-jobs-secret: $JOBS_SECRET" -H "x-sync-mode: catalog"

    In production, call it on a schedule (e.g., Vercel Cron / server cron).

    Digiseller delivery modes

    Per product you can set:

    • MOCK: generates a test key (good for dev)
    • MANUAL: marks delivered with instructions placeholder
    • API: calls DIGISELLER_DELIVERY_ENDPOINT (configure per your seller flow)

    Admin

    • /admin/products: CRUD + Digiseller mapping + featured/trending/popular flags
    • /admin/orders: see order status + fulfillment logs
    • /admin/orders/[id]: manual “Retry” fulfillment

    SEO

    • src/app/sitemap.ts → /sitemap.xml
    • src/app/robots.ts → /robots.txt
    • Product-level OpenGraph metadata via generateMetadata

    QA checklist

    • Browse → add to cart → checkout → Digiseller payment success
    • Digiseller return/callback received → order status becomes FULFILLING
    • Trigger jobs → order becomes DELIVERED, delivery content visible on /order/[id]
    • Fulfillment fail → status becomes FAILED, retry in admin works
    • Access control: another user cannot open /order/[id]
    • RU/EN toggle persists across reloads

    Deploy (Vercel + Railway)

    • Postgres: Railway/Supabase/etc. set DATABASE_URL
    • Next app: Vercel
    • Digiseller: configure return/callback URLs to your deployed /api/digiseller/return and /api/digiseller/callback
    • Cron: call /api/jobs/fulfill every 1–5 minutes with x-jobs-secret

    Notes

    • Delivery payloads are encrypted at rest using DELIVERY_ENCRYPTION_KEY.
    • For production-grade rate limiting, replace the in-memory limiter in src/lib/rate-limit.ts with Redis.
